🛠️ Innovación Técnica: Validación Numérica On-chain

Este contrato evoluciona la arquitectura de los países anteriores al añadir:

Escala de Intensidad (uint8): Implementación de una escala del 0 al 10 para informar al usuario sobre el nivel de picante del plato.

Validación de Rango: Uso de la sentencia require para asegurar que el nivel de picante no exceda el límite permitido (10), garantizando la integridad de los datos.

Estructura de Votación Dual: Funciones inmutables para darLike y darDislike que permiten medir la popularidad de los sabores mexicanos.

Optimización de Gas: Límite estricto de 200 caracteres por registro para asegurar la eficiencia en la red Base.

📖 Modo de Uso

Consultar Menú: En la pestaña Read Contract, usa consultarPlato con un número de ID (ej. 1 para Tacos al Pastor) para obtener los detalles, nivel de picante y votos.

Registrar Plato: En la pestaña Write Contract, usa registrarPlato para añadir nuevas delicias (ej. "Mole Poblano", "Pollo con salsa espesa de cacao y chiles", 5).

Votar: Usa los botones de voto para calificar la ejecución de las recetas.
